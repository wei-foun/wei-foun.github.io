Python种类

​ 众做周知，Python 的底层是由 C 实现的，所以常用的也是 Cpython，除此之外还有 Jpython，lronpython 等

​ Cpython 是官方的版本，在执行时 Cpython 会将 原 (py) 文件 转换成 字节码 (pyc) 文件，然后在 python 虚拟机上执行

​ Jpython 顾名思义就是由 java 实现，执行时会将代码动态编译成 java 字节码，然后在 jvm 上运行

​ IronPython 由 C# 实现，同样执行会将代码编译成 C# 字节码，然后在 CLR 运行

数据类型

​ 常用的七个数据类型：字符串，数字，列表，字典，元组，集合，布尔。其中三个是可变，四个是不可变类型，字符串，数字，元组，布尔 是 不可变数据类型 (可哈希)；列表，字典，集合 是 可变数据类型 (不可哈希)。同时 python 中的 布尔类型 其实是 继承 int 的

数字类型常用方法

​ 数字类型包括了：int，float，long，complex 和 bool，因为 bool 其实是继承了 int ，因此可以用 0 表示 False，1 表示 True，注意点：在 python 自带的命令行解释器中，对于数字的存储有一个叫 “小数据池“ 的东西，用来存储 -5 到 256 的数字以及有规律的字符串，例如 ‘s’ * 20（到 21 的话内存地址就不一样了），只有在规定的范围中，所指向的 id 内存地址才是一致的

​ 但是在 pycharm 中，似乎不会遵守 ‘数据池’ 的规则，相同值的变量，所指向的地址是一致的

​ random 库，是一个内置库，用来产生随机数

​ 补充：decimal 模块支持十进制数的浮点计算，但是不能接收浮点数

​ 补充：字符串和数字之间是允许通过计算符号进行操作的，但是只能是 + 和 *，另外使用 * 时，字符串只能和正整数有效果，之所以 + 和 * 不会出现数字计算是因为 Python 是弱类型的

列表的常用方法

​ 列表是一个可迭代对象的数据类型，可以存储包含 字符串，数字，布尔，字典 等其他的数据类型，当然也可以是自己，也就是列表嵌套列表的结构。列表也是可以通过切片操作来获取元素的，列表的元素和字符串一样，下标索引都是从 0 开始依次往后

​ 注意：使用切片对列表进行操作时，同样如果 下标 超出 len 的长度，是不会抛出 IndexError 的，而是直接返回一个空的 []

​ 补充：

​ 字典的简单排序，通常是在 sort 或 sorted 中的参数 key 使用 lambda 匿名函数来操作，一般情况下都是对一个情况去排序使用 lambda x:x，如果像上面的例子中，要对按照元组中的第二个元素排序的同时，还要求按照如果是第一个元素值是一致的情况下，按照元组的第二个元素排序，就可以使用 lambda t:(t[1], t[0])，这里的两个排序都是升序的，如果要降序排列，就在元素前加一个 - 号

​ 使用 * 号去赋值数组元素时，数组内部嵌套的元素其实都是相同引用，所以当对嵌套内部的元素进行修改时，会造成相互影响

元组常用方法

​ 简单来说元组是一个可读不可改的列表，不可改也就说明元组是可哈希的，并且列表能存储的类型，元组也可以存储，元组内部的元素也可以是不可哈希的，同时元组一样可以通过索引来执行切片获取元素

​ 注意 t = 1,(注意这里有一个,) 这个 t 的类型也是一个元组

​ 元素虽然是不可以变的数据结构，但是任然支持了一些操作，例如 +，可以是两个元组元素合并，产生一个新的元组。* ，可以复制元组的元素，同样会生成一个新元组

字典常用方法

​ 字典是以 key-value 的形式存储数据，是不可哈希的，但是 字典对于 key 的要求是必须可哈希（不可变类型）

​ fromkeys 如果给出第二个参数，则会将当前这个字典的所有 value 都指向这个参数，所以当使用一个可变对象，比如使用列表作为第二参数时，如果要用 key 在列表的添加一个值，会导致当前字典所有 value 的字典中都会添加这个值

集合常用方法

​ 集合是一个无序且数据不能重复的序列，所以可以用来进行去重操作，与字典类似但是没有 value，集合中存储的都是 key，且 key 一样都要是可哈希的

​ 另外，集合虽然看着和数组也很像，但并不支持和数组一样通过下标索引来获取元素

控制语句

if…else 判断

​ 对于简单的 if 条件判断语句，比如比较两个大小返回其中一个，可以使用 三元运算符，但是也要适当使用，如果判断逻辑相对复杂，使用这样的方式，会对代码的可读性有所降低

for 循环

​ 语法： for i in iter: ，从可迭代对象中每一次遍历时，依次向后出去元素

​ 补充：循环判断中，使用 if 语句判断时，多个条件判断可能会使用 if…if…else，也可能是使用 if…elif…else。两种判断方式执行流程是不同的，if…if…else 的流程是，第一个 if 判断不满足条件，会继续判断下一个 if，最后都不满足再执行 else；if…elif…else 的流程是，当第一个 if 判断不满足，那么 elif 不会被触发，直接走到 else

​ if…elif… 的判断更像是一个并列的判断语句，所代表的是一个判断的两种条件，而 if…if… 就是多种不同条件的判断

while 循环

​ 语法： while condition: ，根据条件 condition 判断是否为 True，为 True 则执行内部代码块。但是 while True: 代表的是无限循环，就是死循环，所以循环体内部需要去修改 condition 来退出循环

​ while 的语法中是有判断的， 所以也可以使用 else

文件操作

​ 有两种方式，一种使用 open() 打开文件进行处理，但是最后需要手动将文件关闭，避免一直占用系统资源；另一种常用的方式是使用 with 创建上下文管理器，这种方式在处理完成后，无需手动去关闭，当文件没有被使用的情况下，上下文管理器会自动关闭掉打开的文件

with 上下文管理器

​ 使用 with 操作文件，会自动去触发 with 处理对象的 __enter__ 和 __exit__ 两个方法，当使用 with 创建上下文管理器后，会进入 __enter__ 方法中，之后打开文件和一系列的读写操作才会去执行，如果使用了 as 则会将 __enter__ 方法后的结果给到 as 指向的变量，当最后执行完毕后，会进入 __exit__ 方法中将文件关闭

​ __exit__ 方法实际有三个参数，如果 with 上下文管理器中的代码执行没有错误，那么 __exit__ 的参数值会是三个 None，如果执行出现错误，__exit__ 的参数是 exc_type，exc_value，exc_traceback 各自接收异常信息。对于 __exit__ 方法，如果返回的是 False，会将收到的异常抛出到上下文管理器之外，通过外部的异常捕获来处理，如果是返回 True 则表示不向外抛出异常

​ 补充：

​ 对文件的操作完成后，要关闭文件的目的，是为了将资源回收

​ 首先，对于 Python 的程序而言，默认会去申请一个空间，当操作文件时，会将文件对象变量存在栈空间，实际的文件对象是存在堆空间的

​ 但是，python 程序内存空间的对象，并不能直接对硬盘上的文件进行操作。所以，这中间实际上还有一个部分，就是操作系统的内存，当 python 的文件对象调用 write() 和 read() 时，操作系统会自己划分一块空间，用来执行这个操作

​ 操作系统划分的内存空间，会将硬盘的文件解码出来放入内存，再由 read 读取返回给 python 程序，写入的流程则是方向相反的方式

​ 所以，当 python 程序完成文件操作后，如果不关闭文件，那么操作系统中的那部分内存就不能被回收，导致内存资源的浪费

异常捕获

​ 上面的文件处理就使用了异常捕获来对抛出的异常进行处理，使得程序可以正常执行，而不是被异常阻断

常见的异常类型

异常 描述
NameError 尝试访问一个没有声明的变量等
SyntaxError 语法出现错误，可能就是单词错误或是符号问题
IndexError 下标索引超出序列的范围，循环遍历时或是按照下标取元素会出现这个问题
KeyError 尝试请求一个不存在于字典的主键
AttributeError 尝试访问一个未知的对象属性
ValueError 传递的参数不正确，传递过多或过少的参数等
TypeError 传递的参数类型不被支持进行操作等
ZeroDivisionError 当除数为 0 的错误
… …

​ Python 中的异常全部的基类都是继承于 BaseException，但是根据不同类型还是分成了几类包括：

​ 系统相关的（SystemExit）

​ 按键的输入异常（KeyboardInterrupt）

​ 生成器的（GeneratorExit）

​ 以及最常见的异常（Exception）

​ 大多数程序的异常都是继承于 Exception 的，例如 NameError，OSError，StopIteration 等等很多

主动抛出异常

​ 上面使用过 “万能的异常处理”，即 except Exception as e ，但是这样的处理如果被记录到日志中，却不好排查具体的错误原因，所以通常情况下还是去使用规定的异常名去独自处理各自异常捕获

​ 但是有时候程序可能并不会抛出所感知到的异常，因此可以 手动地抛出异常去进行捕获，即使用 raise 语法

自定义异常

​ 通过上面表里列出的异常和描述，不难看出其实这些异常可能对应的情况有多种，如果想要精确的捕获一种情况的异常错误，可以使用自定义异常，自定义异常的类需要继承 Exception

​ 注意点：在编写自定义的异常时，最好选择继承于 Exception 而不是最终基类的 BaseException

​ 因为捕获父类异常时，继承其的子类的异常也都会被其捕获，所以如果要去捕获 BaseException 的异常，则 KeyboardInterrupt 中的异常同样会被其捕获，那么再去按下 ‘Ctrl + C’ 出来的异常就被自定义的给捕获了，再去使用 ‘Ctrl + C’ 也就没有效果了

not and or

​ 对于 x or y，如果 x 为 真，就返回 x ；否则返回 y

​ 对于 x and y，如果 x 为 假，就返回 x ；如果 x 为 真，y 为 假，返回 y；如果 x 为 真，y 为 真，返回 y

​ 另外，包含了 and 和 or 的逻辑中，and 优先级高于 or

​ 除了之外，not 表示非，not 1 表示为 False，not 0 表示为 True
