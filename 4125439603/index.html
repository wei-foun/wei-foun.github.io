python2 与 python3 的区别

py2 和 py3 的区别

输入与输出的区别

​ 对于输入，python2 中使用的是 raw_input，python3 则直接是 input 函数

​ 对于输出，python2 中 print 是一个关键字，python3 中 print 则是将其改成了函数，所以在 py3 中使用 print 必须加上 ‘ () ‘

​ 补充：如果要在 py2 中想要和 py3 一样使用 print 作为函数来调用，需要需引入

编码不同

​ python2 中使用 ASCII 码，python3 则使用 utf-8，所以可能常见到在 py2 的文件中最上方会有 ＃-*-coding:utf-8-*- 来指明使用 utf-8 编码，py3 则可以不用去主动添加

​ 补充：

字符串存储

​ py2 中如果要定义中文的字符或字符串，需要加上 ‘ u ‘ 的前缀，因为在 py2 中字符串的类型使用的是 unicode，不使用这个前缀的字符串或字符，都是利用字节去储存

​ py3 中字符串的类型直接是 str，并且支持 unicode，所以 py3 可以直接定义中文字符，对于字节来说，py3 使用的前缀是 ‘ b ‘ 来表示；如果要传输中文字符，需要 encode 方法去编码成字节串

计算除法

​ py2 中使用 ‘ / ‘ 做整数除法操作后，得到的结果是会是整型，且会对结果进行截断，只取前面的整数

​ py3 中的整数除法操作，会得到一个 float，如果需要得到整型值，需要使用整除符号 ‘ // ‘

注解提示

​ 这是 py3 做的一个小改进之一，目的是方便 IDE 对参数类型进行检查提示

​ 在 pycharm 中，上面的 ‘8’ 会被用黄色底标记提示参数类型问题，如果去除注解提示，则不会显示这个问题，所以这个注解提示的主要目的是进行提示

​ 补充：这个注解提示并不完全的对类型进行检查和限制，如果需要执行参数类型检查，可以使用第三方的 mypy 库

解包

​ 这是 py2 不支持的， py3 中利用 * 可以对集合参数进行打散解包

限定关键字指明传参

​ 同样是 py3 的一个语法，要求函数中 ‘ * ‘ 之后的参数在传递时，需要指明参数的名字

chained exception

​ 在 py2 中如果需要在一个异常中再去抛出一个异常，那么后者的异常信息栈会被忽略掉，py3 中则更好的支持这个，可以将链式的异常信息栈全部保留，来更好的去进行排错

内置的迭代

​ 在 py2 中内置函数返回列表的，在 py3 中则被改进为返回迭代对象，例如：range，zip，map，dict.values 等等，使用迭代器对象返回的好处自然就是为了节省内存，有其是在大量数据的集合更为明显

​ 另外，py2 中 range 返回列表，如果也想返回迭代器可以使用另一种 xrange，而在 py3 中舍弃了原来返回列表的 range ，而直接使用返回迭代器的 xrange，但是函数名依旧是使用 range

不等于

​ 在 py2 中有一个判断符 <> 表示不等于，在 py3 中则被废弃，统一使用 !=

pyc

​ 相比 py2，在 3 中导入的模块所产生的 pyc 文件都会被统一放入一个文件夹 __pycache__

py3 的新增

yield from

​ 生成器中增加了一个 yield from 语法，来连接子生成器，当然也可以是迭代器或是可迭代对象，可以理解他相比直接用 yield ，可以减少循环嵌套

新增内置库

​ asyncio：这是一个新增的用来处理异步并发的库，提供关键字 async / await 提供简便的方式让协程支持异步编程

​ enum：新增的枚举库，可以得到元素的下标索引

​ concurrent.futures：提供简便接口来实现线程，进程与池等相关的创建

​ 除此之外，py3 还有其他的新增和改进

py2 转 3

​ 1） six 模块

​ 2）2to3 等工具也可以实现代码转换

​ 3） __future__ 模块，可以在 py2 中去支持 py3 的一些实现
