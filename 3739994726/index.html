虚拟化技术

​ 虚拟化技术（Virtualization）是一种资源管理（优化）技术，它将计算机的各种物理资源（例如网络、存储、内存、CPU、GPU等）进行抽象、转换，呈现出可供分割并且任意组合为一个或者多个计算机的配置环境，使用户能够更好的配置和应用计算机硬件资源

​ 按照实现方式分类，分为：硬件虚拟化，软件虚拟化

​ 按照虚拟化程度分为三类：全虚拟化（Full Virtualization）、半虚拟化(Paravirtualization) 和 基于硬件辅助虚拟化（Hardware-assisted virtualization）

​ 云计算层面的虚拟化分为：计算虚拟化、存储虚拟化、网路虚拟化

  • 计算虚拟化

  • 存储虚拟化

  • 网络虚拟化

Hypervisor

​ 一种运行在物理服务器（host machine）和子操作系统虚机（guest machine）之间的中间软件层，可允许多个操作系统和应用共享硬件，也可叫做 VMM（ virtual machine monitor ），即虚拟机监视器，VMware、KVM、Xen、Virtual Box，都属于 Hypervisor

​ Hypervisor 提供了对虚拟机的管理功能，包括虚拟机的创建、启动、停止、删除以及资源的分配和监控。它还确保虚拟机之间的隔离性，使得它们彼此独立运行

​ Hypervisor 有两种类型，type-1 全虚拟化（比如 VMware ESXi），type-2 半虚拟化（比如 VMware workstation）：

  • 直接安装在宿主机的物理硬件上（裸金属虚拟化），虚机对物理硬件调用时需要通过 hypervisor 来接管指令进行操作，这些都依赖于虚拟化层的内核，使得虚机操作系统不会意识到自己是在虚拟环境中

  • 在宿主机操作系统上安装和运行虚拟化程序（基于 Host 虚拟化），优点是实现要更为简单，半虚拟化中由于存在宿主机操作系统，虚机操作系统是可以察觉自己运行在虚拟环境中

Container

​ 容器也是一种虚拟化技术，它不依赖 Hypervisor 虚拟化层，而是直接使用宿主机内核，Linux 容器本质并不是模拟完整的操作系统，通过 docker 引擎层实现上层应用隔离（通过 linux 的 cgroups 和 namespaces 实现隔离），容器的部署和启动速度，以及跨平台兼容都高于虚拟机，但是在安全性和隔离性上相对来是不如虚机的

​ 容器核心技术：资源隔离（cgroup），访问隔离（namespaces），文件系统隔离（rootfs），生命周期控制（容器引擎）

QEMU

​ Quick Emulator，是一款开源的硬件虚拟化软件，可以独立模拟出整台计算机，包括 CPU，内存，IO设备，通过一个特殊的 “重编译器” 对特定的处理器的二进制代码进行翻译，从而具有了跨平台的通用性，具有很高的兼容性

​ Qemu 专注的是全系统仿真（通过软件全模拟硬件），所以更消耗资源，性能也不好

Xen

​ 属于半虚拟化技术，是直接在物理机上通过 Hypervisor 实现资源隔离创建虚机，它直接修改了操作系统代码来支持 xen，通过 doamin0 来管理所有资源

​ Xen 的缺点是操作系统必须进行显式地修改（“移植”）以在 Xen 上运行（但是提供对用户应用的兼容性），所以比较麻烦。使得 Xen 无需特殊硬件支持，就能达到高性能的虚拟化

Hyper-V

​ 是 Microsoft 提出的虚拟化技术，采用微内核的架构，兼顾安全和性能要求。Hyper-V 底层的 Hypervisor 运行在最高的特权级别下，微软将其称为 ring -1（而 Intel 则将其称为 root mode），而虚拟机的 OS 内核和驱动运行在 ring 0，应用程序运行在 ring 3下，这种架构就不需要采用复杂的 BT（二进制特权指令翻译）技术，可以进一步提高安全性

VMware

​ 这个专做虚拟化软件的公司发布有多个虚拟机产品，比如 VMware workstation，VMware ESXi/ESX（企业级）。前者主要是面向 Windows 平台的桌面虚拟化软件，属于 type2 类型的虚拟化。后者属于 type1 虚拟化，直接基于物理机来部署和管理虚拟机

KVM（Kernel-Based Virtual Machine）

​ 基于内核的虚拟机，是 Linux 内核的一个可加载模块，通过调用 Linux 内核，实现对 CPU 的底层虚拟化和内存的虚拟化，使 Linux 内核成为虚拟化层，需要 x86 架构支持虚拟化功能的硬件支持（比如 Intel-VT，AMD-V），是一种全虚拟架构

​ KVM 中虚机被视为常规的 Linux 进程，由标准 Linux 调度程序进行调度。但是 KVM 本身不执行任何硬件模拟，需要用户空间程序通过 /dev/kvm 接口设置一个客户机虚拟服务器的地址空间，向它提供模拟的 I/O，并将它的视频显示映射回宿主的显示屏，这个应用程序是 QEMU

QEMU-KVM

​ KVM 实现了 CPU 和内存的虚拟化，但 kvm 不能虚拟其他硬件设备，因此 qemu 还有模拟 IO设备（磁盘，网卡，显卡等）的作用

​ QEMU 运行在用户空间，通过调用 KVM 在内核空间提供的 /dev/kvm 接口来实现虚拟化。由于 qemu 模拟 io 设备效率不高的原因，现在常常采用半虚拟化的 virtio 方式来虚拟 IO 设备

libvirt

​ 是各种虚拟化平台的虚拟机管理 API 库，一些常用的虚拟机管理工具如 virsh（类似 vim 编辑器），virt-install，virt-manager 等和云计算框架平台（如 OpenStack）底层使用 libvirt 提供的应用程序接口来管理虚机资源

​ libvirt 主要由三个部分组成：API 库，一个守护进程 libvirtd 和一个默认命令行管理工具 virsh

​ 参考：https://mp.weixin.qq.com/s/pDxpuXTMqu0OSR8biviIkg

Kvm 虚机操作

​ 首先需要的是安装好各类工具库

​ 使用 virt-install 创建一台虚机，这个是命令模式。当然也可以在 linux 图形界面通过 virt-manager 来创建，这是 linux 上类似的 VMware 虚机管理软件

​ Kvm 虚机主要包含两个文件：系统配置文件、磁盘镜像文件

​ 系统配置文件以 XML 格式编写，其中包含虚机名，cpu，mem，网卡，磁盘等配置信息，可以通过 virsh edit <domain> 来编辑虚机的 xml 文件配置

​ 磁盘镜像文件（虚机的系统盘），一般有两种格式 raw 和 qcow2。raw 也被称裸磁盘，是纯二进制的格式，性能要更好一些，但占用也相对较大，并且不支持快照。qcow2 是通过 qemu 模拟的虚拟化磁盘，占用空间更小，支持压缩和快照。cow 是 copy on write，当虚机写入块时 qemu/kvm 会对比，将修改的部分复制数据到磁盘镜像
